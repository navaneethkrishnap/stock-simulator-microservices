# 📈 Stock Simulator Microservices

This project is built for **self-learning purposes** to understand and implement **Microservices Architecture** using modern backend technologies.

---

## 🚀 About the Project

The **Stock Simulator Microservices** project simulates a basic stock trading system where users can:

- View stock data
- Buy and sell stocks
- Manage their portfolio
- Handle transactions and balances

The system is designed using **microservices architecture**, where each service is independent and communicates with others.

---

## 🧩 Microservices Included

- **User Service** – Manages user details and account balance  
- **Stock Service** – Provides stock information  
- **Order Service** – Handles buy/sell operations  
- **Portfolio Service** – Tracks user holdings  
- **API Gateway** – Single entry point for all services  
- **Service Registry (Eureka)** – Service discovery  

---

## 🛠️ Tech Stack

- Java
- Spring Boot
- Spring Cloud
- Spring Cloud Gateway
- Eureka Server
- Feign Client
- PostgreSQL
- Maven

---

## 📚 Learning Objectives

- Understand **Microservices Architecture**
- Learn **Service-to-Service Communication**
- Explore **API Gateway & Routing**
- Implement **Service Discovery**
- Handle **Distributed Data Management**
- Understand **Tight vs Loose Coupling**
- Explore **Event-Driven Architecture (future scope)**

---

## ⚙️ How to Run

1. Start **Eureka Server**
2. Start all microservices:
   - User Service
   - Stock Service
   - Order Service
   - Portfolio Service
3. Start **API Gateway**
4. Test endpoints using Postman or browser
